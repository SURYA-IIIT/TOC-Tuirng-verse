MODULE 01

Basic Turing Machine

A mathematical model of computation that defines an abstract machine operating on symbols on a strip of tape according to a table of rules.

📋

Formal Definition

A TM is a 7-tuple: M = (Q, Σ, Γ, δ, q₀, q_accept, q_reject)

  • Q — Finite set of states
  • Σ — Input alphabet (∌ blank)
  • Γ — Tape alphabet (⊇ Σ ∪ {□})
  • δ — Transition function
  • q₀ — Start state
  • q_a — Accept state
  • q_r — Reject state

MODULE 02

Variants of Turing Machine

Despite appearing more powerful, all TM variants are equivalent in computational power to the standard TM.

Multi-Tape Turing Machine

A TM with k tapes, each with its own read/write head. Transition function reads all heads simultaneously and writes/moves independently.

δ: Q × Γᵏ → Q × Γᵏ × {L,R,S}ᵏ

Key Properties:

  • Each tape can be read/written independently
  • Equivalence: Any k-tape TM can be simulated by a single-tape TM
  • Simulation overhead: O(t²) steps where t is original computation steps
  • Tape 1 typically holds input; extra tapes used as work space
✓ Equivalent to Standard TM

2-Tape TM Simulation — aⁿbⁿ

Demonstrating aⁿbⁿ recognition using 2 tapes

Non-Deterministic Turing Machine (NTM)

At each step, the NTM may have multiple possible transitions. It accepts if ANY branch of computation accepts.

δ: Q × Γ → P(Q × Γ × {L,R})

Key Properties:

  • Computation is a tree of possibilities
  • Accepts if any path leads to accept state
  • Equivalent to DTM (simulate using BFS over computation tree)
  • Simulation overhead: exponential in time
✓ Equivalent to Standard TM

NTM Computation Tree

NTM exploring all computation paths

Multi-Track Turing Machine

A single tape divided into multiple tracks. Each cell contains a tuple of symbols, one per track.

Each cell: (a₁, a₂, ..., aₖ) ∈ Γ₁ × Γ₂ × ... × Γₖ

Key Properties:

  • Single tape head reads all tracks simultaneously
  • Useful for storing auxiliary information alongside input
  • Easily convertible to standard TM (encode tuples as single symbols)
  • Common use: track "marked" vs "unmarked" symbols
✓ Equivalent to Standard TM

2-Track TM — Marking Algorithm

Track 1: original input | Track 2: markers

Two-Way Infinite Tape TM

Tape is infinite in both directions (−∞ to +∞) instead of being bounded on the left.

Tape: ... □ □ a₁ a₂ ... aₙ □ □ ...

Key Properties:

  • Head can move left indefinitely (no left boundary)
  • Simulation: fold the tape — upper half for positive positions, lower for negative
  • Simulated by 2-track TM with standard semi-infinite tape
  • More natural model for some algorithms
✓ Equivalent to Standard TM

TM as Computer of Integer Functions

TM can compute integer functions f: ℕᵏ → ℕ by encoding integers in unary (n = 1ⁿ) or binary.

Input: 1ⁿ (n ones) → Compute f(n) → Output: 1^f(n)

Computable Functions:

  • Addition: f(m, n) = m + n
  • Multiplication: f(m, n) = m × n
  • Monus: f(m, n) = max(m−n, 0)
  • Any primitive recursive function

Encoding:

Multiple args separated by 0: 1^m 0 1^n represents (m, n)

MODULE 03

Universal Turing Machine

A UTM simulates any TM M on any input w by reading the encoding ⟨M, w⟩ from its tape — the theoretical foundation of modern computers.

🌐

What is a UTM?

The Universal Turing Machine U takes as input the encoding ⟨M⟩ of any TM M and an input w, and simulates M on w.

U accepts ⟨M, w⟩ ⟺ M accepts w

This is the basis of stored-program computers — the program (TM) is just data on the tape!

📦

TM Encoding

To encode TM M = (Q, Σ, Γ, δ, q₀, q_a, q_r):

  • States: q₁, q₂, ..., qₙ (q₁ = start)
  • Symbols: 0→X₁, 1→X₂, □→X₃
  • Directions: L→D₁, R→D₂
  • Each rule encoded in binary
  • Rules separated by 11, encoding by 111
Example encoding fragment:
0i 1 0j 1 0k 1 0l 1 0m
q_i reads X_j → write X_k, move D_l, go to q_m

MODULE 04

Languages & TM Acceptance

Understanding what kinds of problems Turing Machines can solve — and their fundamental limits.

Chomsky Hierarchy

✅

Recursive Languages

Decidable

A language L is recursive (decidable) if there exists a TM M such that:

  • M accepts every w ∈ L
  • M rejects every w ∉ L
  • M always halts on all inputs
L is recursive ⟺ L and L̄ are both RE

Examples:

aⁿbⁿ Equal a's and b's ▶ Test
Palindromes Reads same forwards & backwards ▶ Test
aⁿbⁿcⁿ Equal counts of a, b, c ▶ Test
{ww^R} Concatenation with reverse ▶ Test
⚡

Recursively Enumerable

Semi-Decidable

A language L is recursively enumerable (RE) if there exists a TM M such that:

  • M accepts every w ∈ L
  • M may reject OR loop forever for w ∉ L
  • M may not halt on all inputs
Recursive ⊊ RE

Examples:

Halting Problem L_HP = {⟨M,w⟩ : M halts on w} RE but not Recursive
L_u Universal language {⟨M,w⟩ : M accepts w} RE but not Recursive
Diophantine Eqs Solvable integer polynomial equations RE
Non-RE: L̄_HP (complement of halting problem) — no TM can even semi-decide this!

MODULE 05

Church-Turing Thesis

The fundamental conjecture connecting intuitive computability to formal TM computation.

"

Every effectively computable function is computable by a Turing Machine.

— Alonzo Church & Alan Turing, ~1936

What It Claims

Any function that can be computed by any algorithmic means (human, computer, any model) can also be computed by a Turing Machine.

Why It's a Thesis, Not a Theorem

It cannot be formally proved because "effectively computable" is an intuitive, informal concept. It's supported by evidence: every known model of computation is TM-equivalent.

Equivalent Formulations

  • λ-calculus (Church)
  • μ-recursive functions (Kleene)
  • Post correspondence systems
  • RAM machines, C programs
  • Cellular automata

Implications

  • Sets a limit on what computers can do
  • Undecidable problems are absolutely unsolvable
  • No super-Turing computer can exist
  • Justifies using informal algorithms as TMs
